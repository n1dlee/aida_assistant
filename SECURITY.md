# Security Policy

## Supported Versions

The following versions of AIDA Assistant are currently supported with security updates:

| Version | Supported |
|---------|----------|
| main    | ✅        |
| older versions | ❌ |

We recommend always using the latest version of the project.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not open a public issue.**

Instead, send a detailed report to:

`zayniddinovnodir@gmail.com`

Please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggestions for mitigation (if available)

## Response Process

After receiving a report, we will:

1. Acknowledge receipt within 48 hours
2. Investigate the issue
3. Work on a fix if the vulnerability is confirmed
4. Release a patch as soon as possible

## Disclosure Policy

- We ask reporters to avoid public disclosure until the issue is resolved
- Once fixed, we may disclose the vulnerability publicly with proper credit

## Scope

This policy applies to all parts of the AIDA Assistant project, including:

- Core system logic
- API endpoints
- Integrations
- Data handling and storage

## Notes

Security is a shared responsibility. Contributions that improve the safety, reliability, and integrity of the system are always appreciated.
